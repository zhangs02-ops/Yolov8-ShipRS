# YOLOv8-ShipRS v2 修改说明文档

## 一、改进概述

基于 v1（P2 + CoordAttention + C2f_CA + AdaptiveDetect），针对老师提出的**"底层特征引出 + 特征融合"**建议，结合 2025 年最新 YOLOv8 小目标检测论文，在 v2 中实施了以下改进：

| 改进项                             | 作用                                                   | 论文参考                             |
| ---------------------------------- | ------------------------------------------------------ | ------------------------------------ |
| **渐进式特征融合（Neck架构重构）** | 替代传统PAFPN跨层拼接，逐级融合避免信息冲突            | AFPN (2023), YOLOv8s-Improved (2025) |
| **EMA 高效多尺度注意力**           | 三分支并行提取多尺度空间特征，无额外参数开销           | EMA (arXiv:2305.13563)               |
| **Shape-IoU 损失函数**             | 对非方形目标（舰船长宽比变化大）更敏感的边界框回归损失 | Shape-IoU (arXiv:2312.16253)         |

## 三、各改进项详细说明

### 3.1 渐进式特征融合（Neck 架构重构）

**问题**：传统 PAFPN 在跨层融合时，P3 直接从 Backbone P3 和 FPN P4 上采样拼接，跨层信息差异大，容易产生特征冲突。

**解决方案**：渐进式融合——每一层只与相邻层融合，信息逐级传递。

#### v1 (传统 PAFPN):

```
FPN: P5→P4→P3→P2（每层直接从Backbone取特征拼接）
PAN: P2→P3→P4→P5
```

#### v2 (渐进式融合):

```
FPN: P5 →[融合P4_backbone]→ P4_fused
      P4_fused →[融合P3_backbone]→ P3_fused
      P3_fused →[融合P2_backbone]→ P2_fused

PAN: P2_fused →[融合P3_fused]→ P3_enhanced
      P3_enhanced →[融合P4_fused]→ P4_enhanced
      P4_enhanced →[融合P5+SPPF]→ P5_enhanced
```

**关键区别**：

- v1 的 P3 融合层直接拼接 Backbone P3 + 上采样的原始 P4
- v2 的 P3 融合层拼接 Backbone P3 + **已经过 P5→P4 融合的 P4_fused**

这确保了语义信息逐级向底层传递，避免了跨层信息冲突。

**修改位置**：`yolov8-ShipRS-v2.yaml` head 部分

---

### 3.2 EMA 高效多尺度注意力

**动机**：v1 使用的 CoordAttention 虽然效果好，但 EMA 在多个论文中展现了更优的性能/效率比。

**EMA 原理**：

1. 将通道分组，减少计算量
2. 三分支并行：1D水平池化 + 1D垂直池化 + 1x1卷积
3. 跨空间信息交互：两个分支的全局平均池化权重互相乘
4. Sigmoid 注意力加权输出

**与 CoordAttention 的对比**：

| 特性       | CoordAttention | EMA              |
| ---------- | -------------- | ---------------- |
| 分支数     | 2（H + W）     | 3（H + W + 1x1） |
| 跨空间交互 | 无             | 有（双分支交叉） |
| 参数开销   | 低             | 同等低           |
| 多尺度能力 | 单尺度         | 内建多尺度       |

**修改位置**：

- `conv.py`: 新增 `EMA` 类
- `block.py`: 新增 `BottleneckEMA` 和 `C2f_EMA` 类

---

### 3.3 Shape-IoU 损失函数

**动机**：舰船目标的长宽比变化大（从小艇的1:1到货轮的5:1），标准 CIoU 对形状差异不够敏感。

**Shape-IoU 原理**：

1. 计算预测框和真实框的宽高比差异
2. 对宽和高分别计算形状惩罚项（非单调，差异越大惩罚越大）
3. 结合中心距离惩罚和形状惩罚
4. 最终损失 = IoU - 中心距离惩罚 - 形状惩罚

**与 CIoU/NWD 的对比**：

| 损失函数        | 关注点                             | 适合场景           |
| --------------- | ---------------------------------- | ------------------ |
| CIoU            | 重叠面积 + 中心距离 + 宽高比       | 通用               |
| NWD             | 高斯分布距离                       | 极小目标           |
| Shape-IoU       | 重叠面积 + 中心距离 + **形状差异** | 非方形目标（舰船） |
| NWD + Shape-IoU | 两者结合                           | 极小非方形目标     |

**修改位置**：

- `metrics.py`: 新增 `bbox_shape_iou()` 函数
- `loss.py`: `BboxLoss.forward()` 支持 `use_shape_iou` 参数
- `default.yaml`: 新增 `shape_iou` 配置项

---

### 3.4 ASFF 自适应空间特征融合（备用模块）

在 `block.py` 中新增了 `ASFF` 类，作为可选的特征融合模块。ASFF 让网络自动学习每个尺度特征的重要性权重，解决传统 concat/add 的特征冲突问题。

**当前未在 YAML 中启用**，可在后续消融实验中测试。

## 七、使用方法

### 训练 v2 模型（Shape-IoU）

```bash
python yolov8_shipRS_v2.py
```

### 训练 v2 模型（NWD）

```bash
python yolov8_shipRS_v2_nwd.py
```

### 训练消融模型（仅EMA）

```bash
python yolov8_EMA.py
```

### 代码中切换损失函数

```python
model = YOLO("yolov8-ShipRS-v2.yaml")
model.train(
    data="seaship.yaml",
    shape_iou=True,   # 使用 Shape-IoU
    # nwd=True,       # 或使用 NWD（与 shape_iou 互斥，nwd 优先）
    # nwd_c=2.0,
    ...
)
```
