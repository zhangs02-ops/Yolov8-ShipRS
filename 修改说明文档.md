# YOLOv8 船舶遥感小目标检测改进方案 — 修改说明文档

> 基于 Ultralytics YOLOv8 (v8.4.21)，针对船舶遥感图像中的小目标检测场景进行系统性增强。

---

## 一、项目概述

- **基础框架**: Ultralytics YOLOv8 v8.4.21
- **目标任务**: 遥感图像船舶小目标检测
- **数据集**: Airbus Ship Detection（YOLO 格式，1 类：ship）
- **训练参数**: 500 epochs, SGD, lr0=0.01, batch=64, imgsz=640, fraction=0.3, patience=30

---

## 二、改进内容总览

本项目共进行了 **两轮渐进式改进**：

| 轮次 | 改进方向 | 核心内容 |
|:---:|---|---|
| 第一轮 | 模型架构增强 | P2 小目标层 + 坐标注意力 (CA) + 自适应检测头 (AdaptiveDetect) → ShipRS |
| 第二轮 | 损失函数优化 | NWD (Normalized Wasserstein Distance) 替代 CIoU |

---

## 三、第一轮改进 — 模型架构增强

### 3.1 P2 小目标检测层

**目的**: 在原始 3 尺度检测 (P3/P4/P5) 基础上增加 P2 层，提升对小目标的感知能力。

- **P2 特征图尺寸**: 160×160（输入 640×640 时），步幅为 4
- **效果**: 4 倍于 P3 的空间分辨率，能捕获更细粒度的小目标特征
- **配置文件**: `ultralytics/cfg/models/v8/yolov8-P2.yaml`

### 3.2 坐标注意力机制 (Coordinate Attention)

**目的**: 引入位置敏感的注意力机制，增强小目标定位精度。

#### 3.2.1 CoordAttention 模块

**文件**: `ultralytics/nn/modules/conv.py`

将通道注意力分解为 H 方向和 W 方向两个一维特征编码，精确捕获空间位置信息：

```
输入 → H-Pool(B,C,H,1) + W-Pool(B,C,1,W)
     → 拼接 → 共享变换(Conv1×1 + BN + SiLU)
     → 拆分 → H-Attn(Sigmoid) × W-Attn(Sigmoid)
     → 输入 × H-Attn × W-Attn
```

- **参考文献**: Coordinate Attention for Efficient Mobile Network Design (CVPR 2021)
- **参数开销**: 约 2-4%，无推理速度下降

#### 3.2.2 BottleneckCA 与 C2f_CA

**文件**: `ultralytics/nn/modules/block.py`

- **BottleneckCA**: 标准 Bottleneck 末端接入 CoordAttention
- **C2f_CA**: 将 C2f 中所有 Bottleneck 替换为 BottleneckCA
- **部署位置**: 骨干网络 P3/P4 层 + 检测头 P2/P3 路径（小目标关键通路）
- **配置文件**: `ultralytics/cfg/models/v8/yolov8-CA.yaml`

### 3.3 自适应检测头 (AdaptiveDetect)

**目的**: 为不同尺度特征图提供差异化的特征增强，小目标获得更强的增强。

#### 3.3.1 ECA 高效通道注意力

**文件**: `ultralytics/nn/modules/conv.py`

- 使用 1D 卷积替代全连接层实现通道注意力
- 核大小自适应计算: `k = |log₂(C)/γ + b/γ|`（取奇数）
- **参考文献**: ECA-Net (CVPR 2020)

#### 3.3.2 FRM 特征精炼模块

**文件**: `ultralytics/nn/modules/head.py`

每个检测尺度配备独立的 FRM，结构如下：

```
输入 → 膨胀深度可分离卷积 → BN → 逐点卷积 → ECA → ×scale_weight → +残差
```

**尺度感知增强策略**（核心设计）：

| 尺度 | 膨胀率 | 感受野 | 增强权重 | 含义 |
|:---:|:---:|:---:|:---:|---|
| P2 (scale_idx=0) | 3 | 7×7 | 1.50 | 最强增强，检测微小船舶 |
| P3 (scale_idx=1) | 2 | 5×5 | 1.33 | 较强增强，小型船舶 |
| P4 (scale_idx=2) | 1 | 3×3 | 1.17 | 适度增强，中型船舶 |
| P5 (scale_idx=3) | 1 | 3×3 | 1.00 | 基准增强，大型船舶 |

#### 3.3.3 AdaptiveDetect 检测头

**文件**: `ultralytics/nn/modules/head.py`

- 继承标准 Detect 头，在推理前对每个尺度执行 FRM 预处理
- 即插即用，完全兼容训练/推理/导出流程
- **配置文件**: `ultralytics/cfg/models/v8/yolov8-AdaptiveDetect.yaml`

### 3.4 ShipRS 完整架构

**配置文件**: `ultralytics/cfg/models/v8/yolov8-ShipRS.yaml`

将上述三个改进组合为完整的船舶检测架构：

```
┌─────────────────── Backbone ───────────────────┐
│ Conv(64,s=2) → Conv(128,s=2) → C2f(128)       │  ← P2/4 小目标入口
│ → Conv(256,s=2) → C2f_CA(256) ★               │  ← P3/8 坐标注意力
│ → Conv(512,s=2) → C2f_CA(512) ★               │  ← P4/16 坐标注意力
│ → Conv(1024,s=2) → C2f(1024) → SPPF(1024)     │  ← P5/32
└────────────────────────────────────────────────┘
           ↓
┌──────────────────── Head ─────────────────────┐
│ FPN: P5 → P4 → P3(C2f_CA★) → P2(C2f_CA★)    │  ← 自顶向下融合
│ PAN: P2 → P3 → P4 → P5                        │  ← 自底向上融合
│ → AdaptiveDetect(P2, P3, P4, P5) ★            │  ← 4 尺度自适应检测
└────────────────────────────────────────────────┘
```

## 四、第二轮改进 — NWD 损失函数

**目的**: 用 Normalized Wasserstein Distance 替代 CIoU 作为边界框回归损失，减少小目标定位对 IoU 的敏感性。

### 4.1 原理

CIoU 的核心问题：小目标的微小偏移会导致 IoU 剧烈变化，使损失信号不稳定。

NWD 将边界框建模为二维高斯分布，计算分布间的 Wasserstein 距离：

```
W₂² = ||μ₁ - μ₂||² + ||Σ₁^(1/2) - Σ₂^(1/2)||_F²

NWD = exp(-√W₂ / C)
```

- `C` 为归一化常数（默认 2.0），越大越宽容
- 输出范围 [0, 1]，可直接替代 IoU 参与损失计算
- 对小目标的微小位移更加鲁棒

### 4.2 修改文件清单

| 文件 | 修改内容 |
|---|---|
| `ultralytics/utils/metrics.py` | 新增 `bbox_nwd()` 函数 |
| `ultralytics/utils/loss.py` | `BboxLoss` 类增加 `use_nwd`/`nwd_c` 参数，前向传播中切换 NWD/CIoU |
| `ultralytics/utils/loss.py` | `v8DetectionLoss` 类传递 NWD 参数至 BboxLoss 和 TaskAlignedAssigner |
| `ultralytics/utils/tal.py` | `TaskAlignedAssigner` 类增加 `iou_calculation()` 方法，支持 NWD 度量 |
| `ultralytics/cfg/default.yaml` | 新增配置项 `nwd: False` 和 `nwd_c: 2.0` |
| `ultralytics/cfg/__init__.py` | 注册 `nwd` (bool) 和 `nwd_c` (float) 参数验证 |

### 4.3 使用方式

```python
model.train(data="seaship.yaml", nwd=True, nwd_c=2.0, ...)
```

设置 `nwd=True` 后，边界框回归损失和标签分配的 IoU 计算均自动切换为 NWD。

## 五、消融实验设计

为验证各改进的独立贡献，设计了以下对照实验：

| 实验 | 模型配置 | 训练脚本 | 改进点 |

## 八、训练参数详细说明

所有实验使用统一的训练参数，以确保消融实验的公平对比。

### 8.1 显式设置的参数

以 `yolov8_shipRS.py` 为代表，所有训练脚本传入以下参数：

```python
model.train(
    data="seaship.yaml",       # 数据集配置
    epochs=500,                # 训练轮数（默认100）
    fraction=0.3,              # 使用30%训练数据（默认1.0）
    optimizer='SGD',           # 随机梯度下降优化器（默认auto）
    lr0=0.01,                  # 初始学习率
    lrf=0.01,                  # 最终学习率 = lr0 × lrf = 0.0001
    momentum=0.937,            # SGD 动量
    weight_decay=0.0005,       # L2 正则化系数
    warmup_epochs=3,           # 学习率预热轮数
    batch=64,                  # 批大小（默认16）
    imgsz=640,                 # 输入图像尺寸
    patience=30,               # 早停耐心值（默认100）
    seed=0,                    # 随机种子，保证可复现
    device=0,                  # 使用 GPU 0
    workers=8,                 # 数据加载线程数
)
```

NWD 变体额外传入 `nwd=True, nwd_c=2.0`。

### 8.2 使用默认值的关键参数

#### 训练策略

| 参数 | 默认值 | 说明 |
|---|:---:|---|
| `pretrained` | `True` | 使用 COCO 预训练权重初始化（迁移学习） |
| `amp` | `True` | 自动混合精度训练 (FP16)，加速训练并节省显存 |
| `cos_lr` | `False` | 使用线性学习率衰减（非余弦退火） |
| `deterministic` | `True` | 确定性模式，结果可复现 |
| `rect` | `False` | 不使用矩形批次，所有图像 resize 为 640×640 |
| `close_mosaic` | `10` | 最后 10 个 epoch 关闭 mosaic 增强 |
| `multi_scale` | `0.0` | 不使用多尺度训练 |
| `single_cls` | `False` | 非单类模式（数据集仅 1 类，效果等同） |
| `cache` | `False` | 不缓存图像到内存 |
| `save_period` | `-1` | 不额外保存中间检查点（仅保存 best 和 last） |

### 8.3 学习率调度总结

```
lr
0.01 ┤━━━╮
     │   ╲
     │    ╲  线性衰减 (cos_lr=False)
     │     ╲
0.0001┤━━━━━╲━━━━━━━━━━━━━━━━━━━━━━━━━
     └──┬───┬──────────────────────┬──→ epoch
        0   3                    500
        │warmup│     训练阶段      │
```

- 预热阶段 (epoch 0→3): 学习率从 ~0 线性升至 `lr0=0.01`
- 训练阶段 (epoch 3→500): 从 `lr0=0.01` 线性衰减至 `lr0×lrf=0.0001`
- 早停 (`patience=30`): 若验证指标连续 30 个 epoch 无提升则提前终止
